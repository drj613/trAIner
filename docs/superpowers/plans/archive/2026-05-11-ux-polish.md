## Task 9: Fix "Resetting…" Stuck Loading State on Workspace Reset

**Files:**
- Modify: `src/lib/backup/backup.ts`
- Modify: `src/components/app/SettingsClient.tsx`
- Test: `src/lib/backup/backup.test.ts`

**Root cause:** `resetWorkspace()` calls `indexedDB.deleteDatabase(DB_NAME)` while the database has an open cached connection (the `IDBPDatabase` instance cached by `getDb()`). `deleteDatabase` blocks until all open connections close, so `req.onsuccess` never fires. `resetDbConnection()` is called inside `onsuccess` — but since `onsuccess` never fires, the connection never closes, so the delete never completes. `await resetWorkspace()` hangs forever, `window.location.reload()` is never called, and the button stays "Wiping…" indefinitely.

**Fix:** Call `resetDbConnection()` *before* `indexedDB.deleteDatabase()` to close the cached connection first. Also handle the `onblocked` event (triggered when another tab has the DB open) so the promise rejects cleanly instead of hanging. Surface the error in `SettingsClient` so users see what went wrong.

```typescript
// Before:
export async function resetWorkspace(): Promise<void> {
  return new Promise((resolve, reject) => {
    const req = indexedDB.deleteDatabase(DB_NAME);
    req.onsuccess = () => { resetDbConnection(); resolve(); };
    req.onerror = () => reject(req.error);
  });
}

// After:
export async function resetWorkspace(): Promise<void> {
  resetDbConnection(); // close cached connection first — deleteDatabase blocks on open connections
  return new Promise((resolve, reject) => {
    const req = indexedDB.deleteDatabase(DB_NAME);
    req.onsuccess = () => resolve();
    req.onerror = () => reject(req.error);
    req.onblocked = () =>
      reject(new Error("Reset blocked — close other trAIner tabs and try again."));
  });
}
```
