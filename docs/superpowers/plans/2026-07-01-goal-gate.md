# Goal-Gate Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A per-program training goal (`general | hypertrophy | strength | endurance | other`) gates which analysis dimensions count toward the A–F grade; gated-out dimensions render as reference-only, the grade renormalizes over what remains and is labeled "partial", and a mismatch nudge flags goal↔content disagreement.

**Architecture:** The goal is a **gating input, never a prescription input** — no per-goal landmarks, no re-scoring. One gate table in `thresholds.ts` selects graded dimensions; `computeOverallScore` renormalizes weights over that subset; `analyze.ts` filters findings to graded dimensions and emits `goalScope` + info-level `notes`; display threads it through; UI adds a goal `<select>` on the analysis card, a profile default, and creation/import defaulting. Spec: `docs/superpowers/specs/2026-07-01-goal-gate-design.md` — read it first.

**Tech Stack:** TypeScript, React, Jest via `bun run test <path>` (jest does NOT type-check — run `bunx tsc --noEmit` for that; test files are excluded from tsc, so runtime shape of test literals matters, compile shape doesn't), Vite (`bun run build` type-checks app code).

**PREREQUISITE:** The branch `fix/analysis-residual-findings` must be merged first. This plan assumes its post-merge state: `toDisplayAnalysis.ts` computes `daysPerWeek` and exports `MUSCLE_LABEL`; `src/lib/analysis/sheetPrompt.ts` exists; `periodization.ts` has set-weighted `weekIsHeavy`; session display maps red warnings to "bad". Verify before starting: `test -f src/lib/analysis/sheetPrompt.ts && grep -q "export const MUSCLE_LABEL" src/lib/analysis/toDisplayAnalysis.ts && echo READY`.

- [ ] **Step 6: Implement renormalized scoring**

In `src/lib/analysis/score.ts`, add `DimensionKey` to the type-import from `./types`, then replace `computeOverallScore`:

```ts
const ALL_DIMENSIONS: readonly DimensionKey[] = ["volume", "session", "balance", "periodization"];

export function computeOverallScore(
  dimensions: {
    volume: DimensionScore;
    session: DimensionScore;
    balance: DimensionScore;
    periodization: DimensionScore;
  },
  graded: readonly DimensionKey[] = ALL_DIMENSIONS,
): DimensionScore {
  const w = DIMENSION_WEIGHTS;
  let weighted = 0;
  let totalWeight = 0;
  for (const key of graded) {
    weighted += dimensions[key].score * w[key];
    totalWeight += w[key];
  }
  const score = totalWeight > 0 ? Math.round(weighted / totalWeight) : 0;
  return { name: "Overall", score, grade: scoreToGrade(score) };
}
```

- [ ] **Step 2: Extract heavySetShare**

In `src/lib/analysis/periodization.ts`, replace `weekIsHeavy` with:

```ts
/** Set-weighted share of heavy work (≥85% 1RM / ≤3RM / RPE ≥ 9 / reps ≤ 3). */
export function heavySetShare(days: ProgramDay[]): number {
  let totalSets = 0;
  let heavySets = 0;
  for (const exercise of weekExercises(days)) {
    const sets = getEffectiveSets(exercise);
    totalSets += sets;
    if (exerciseIsHeavy(exercise)) heavySets += sets;
  }
  return totalSets === 0 ? 0 : heavySets / totalSets;
}

function weekIsHeavy(weekDays: ProgramDay[]): boolean {
  // Weigh by sets, not exercise count: in a volume-cut final week the heavy
  // main lifts carry most sets even when accessories outnumber them.
  return heavySetShare(weekDays) >= 0.5;
}
```

(This is a pure extraction — the existing `weekIsHeavy` behavior and its tests must not change.)
